  # 동기 비동기
   => 동작(모션)을 제어할 때 사용
   => event-driven (이벤트 주도 방식) -> 이벤트가 일어났을 때 무언가 열심히 한다
   => 눈에 보이지 않음 -> 들여다 보는 일련의 과정이 필요
   ===> 실행을 제어하는 것
   => 함수는 최상급 취급을 받기 때문에 언제든지 콜링이 가능함
   => 순서를 어기는 setTimeout() -> 먼저 완료된 것 부터 바로 실행 (차례가 없음) : 비동기 
   * setTimeout 이나 setInterval의 경우 각자의 방식이 있기 때문에 순서를 조심 해야 함 
    ===> 함수안에 함수를 넣어서 해경 -> 콜백 지옥
  # Promise : 약속
   => 실행을 일단 멈추고 호출 순서를 작성한대로 나올 수 있게 함 
    * pending : 대기하다(실행을 멈춘다)
    ==> 비동기 방식을 취하고 있는 setTimeout() / setInterval() 에 Promise를 달면 동기방식으로 실행되기 시작함
    ==> 뒤죽박죽 움직이는 함수를 핸들링 하기 편해짐
    ==> 단점 : returen , new Promise(), 콜백함수, resolve, reject와 같은 함수를 써야 하는 상황(복잡함)
   => Promise().then().then().then().then().then().
   + then : 그러고 나서~

   ! return에 약속을 달면 순서를 지킴
   ! then()이 붙은 순서대로 순서를 지키게끔 만들 수 있음
   ! 내가 순서를 많이 써야 하는 상황에서 떠올릴 키워드는 'promise'

   => pending : 대기
   => fulfilled(resolve) : 성공
   => reject : 실패

   + 비동기 방식 : 병렬 처리(동시 실행)
   + 동기 방식 : 직렬 처리(순서대로 실행)

   * node.js : 논블록킹 방식의 I/O input/output
   논블록킹 -> 비동기 방식
   입력출력이 매우 좋은 -> 순서를 어기고 먼저 끝나는 대로 처리해주는 방식
